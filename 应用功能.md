# 邮箱简历投递应用功能

## 应用概述

- 一个类似淘宝“购物车”模式的邮箱简历自动投递应用。
- 帮助用户批量管理和投递基于不同渠道收集到的招聘信息。


## 核心功能模块

### 招聘信息获取模块

**信息来源渠道：**

- 付费服务接入：对接定期整理实习/招聘信息的第三方服务。
- 爬虫系统：从公众号自动爬取招聘信息。
- 用户手动导入：用户复制粘贴学校或其他渠道的招聘信息。

**处理流程：**

- 对原始文本做结构化提取，解析出关键信息字段。
- 关键字段示例：公司名称、岗位名称、岗位要求、投递邮箱、截止日期、工作地点等。
- 将结构化结果存入数据库，供前端展示和后续投递使用。


### 简历投递管理模块（“购物车”）

- 用户可以像在淘宝选商品一样浏览岗位列表，将想投递的岗位加入“投递购物车”。
- 用户也可以导入自己从其他渠道获取的招聘信息并加入购物车。
- 支持对购物车中的岗位进行查看、编辑、删除和批量选择。


### 自动化投递模块

**基于岗位要求和用户信息的自动定制：**

- 自动根据岗位要求和规则生成规范的文件名，例如：`姓名_岗位_公司_简历.pdf`。
- 自动生成邮件标题，支持按不同公司或学校要求配置标题模板。
- 根据预设模板 + 用户填写的个人信息 + 岗位描述，自动生成或填充个性化自荐信内容。
- 对购物车中选中的岗位进行批量邮件投递。

**用户信息管理：**

- 用户维护个人资料：姓名、学校、专业、年级、联系方式、技能等基础信息。
- 用户上传多份简历文件（通用版、岗位定制版等）并可选择投递时使用哪一份。
- 用户维护自荐信模板和可复用片段，用于自动填充邮件正文。


## 技术开发链路

### 阶段一：数据获取层

- 开发公众号爬虫程序，解决登录、分页、反爬等问题，对内容进行初步清洗。
- 对接第三方实习/招聘信息服务，设计定时同步或回调机制。
- 提供用户文本粘贴/上传接口，对用户输入的招聘信息做基本解析。


### 阶段二：数据处理层

- 使用规则或 NLP 将原始文本解析为结构化字段（公司、岗位、要求、邮箱等）。
- 做数据校验、去重、归一化处理（如统一岗位名称、地点格式）。
- 设计并实现数据库表：招聘信息表、来源渠道表等。


### 阶段三：业务逻辑层（优化版）

**投递清单管理：**

- 建立投递任务单机制，每个任务单包含目标岗位、关联简历版本、AI 生成的自荐信。
- 预检机制：批量投递前自动校验（简历文件大小、邮箱格式、用户回信地址设置）。
- 去重过滤：自动对比历史记录，防止 7 天内向同一公司同一邮箱重复投递。

**AI 智能适配引擎：**

- 动态指纹消除：对模板生成 5-10 个表述不同但意思相同的版本，随机分配防止被判定垃圾邮件。
- 附件名脱敏优化：根据岗位关键词自动命名，如 [姓名]-应聘[岗位名]-[学校]-[专业].pdf。
- 变量增强系统：支持高级占位符 {{company_name}}、{{job_title}}、{{source}}、{{user_highlights}}。
- 智能附件管理：自动判断使用 PDF 附件还是长图文正文，超大简历转为云端私密链接。

**云端推送服务：**

- 阿里云 API 深度集成：动态设置回信地址、AI 自动提取用户姓名设置发信人昵称。
- 异步调度系统：使用 Redis/RabbitMQ 存储投递任务，多线程消费，针对不同收件域设置发送频率。
- 全链路状态追踪：监听阿里云回调信号，区分提交成功、已投递、被拒收、被拦截等状态。
- 失败重试策略：网络波动导致失败采用指数退避重试，被拦截类失败自动停止并提醒用户。


### 阶段四：前端展示层

- 岗位列表页：展示各渠道整合后的岗位信息，支持筛选和搜索。
- 购物车页面：展示已选岗位，支持批量勾选、移除、发起投递。
- 用户中心：维护个人信息、简历文件、自荐信模板和投递设置。
- 投递记录页：展示历史投递记录及状态，方便用户跟踪。


### 阶段五：测试与优化

- 功能与链路测试：从导入岗位到批量投递的全流程联调。
- 邮件送达测试：不同邮箱服务商（如 QQ、163、Gmail 等）的送达效果测试和限速策略。
- 性能与稳定性优化：大批量岗位和投递场景下的性能与错误处理。
